# Avito Frontend Trainee Assignment

Фронтенд-часть тестового задания для Avito с отдельным backend-сервисом на Fastify.

Проект реализует:
- список объявлений с поиском, фильтрами, пагинацией и сортировкой;
- страницу объявления с признаком `needsRevision`;
- страницу редактирования объявления;
- AI-помощники для генерации описания и оценки цены;
- локальное автосохранение черновика формы;
- запуск фронта и бэка через `docker compose`.

## Стек

### Frontend
- React 19
- TypeScript
- Vite
- React Router
- TanStack Query
- MUI

### Backend
- Node.js
- Fastify
- Zod

### AI
AI-функции работают через внешний LLM API. В текущей конфигурации backend отправляет запросы в OpenAI-compatible Chat Completions / Completions endpoint с моделью, заданной через `AI_MODEL`. В проекте это используется для:
- генерации описания объявления;
- оценки рекомендованной цены с коротким обоснованием.

## Что было реализовано

### 1. Список объявлений
- поиск по названию;
- фильтр по категориям;
- фильтр `Требуют доработки`;
- пагинация;
- переключение вида карточек `grid/list`;
- сортировка по названию, дате создания и цене.

Важно: сортировка по цене была добавлена и на frontend, и на backend.  
На стороне сервера в [server.ts](C:\Users\dori-\OneDrive\Desktop\projects\avito-back\server\server.ts) добавлена обработка `sortColumn === 'price'`, чтобы сортировка действительно работала на API, а не только в интерфейсе.

### 2. Страница объявления
- отображение основных полей объявления;
- отображение характеристик по категории;
- отображение описания;
- индикация `needsRevision`;
- список незаполненных или проблемных полей для объявления, требующего доработки.

### 3. Редактирование объявления
- форма редактирования категории, названия, цены, характеристик и описания;
- счётчик символов у описания;
- сохранение объявления через API;
- уведомления об успешном сохранении и ошибках;
- валидация обязательных полей.

Для обязательных полей (`Категория`, `Название`, `Цена`) реализована клиентская валидация:
- пустые поля подсвечиваются красной рамкой;
- под полем показывается сообщение вида `Название должно быть заполнено`.

### 4. AI-функции
- кнопка генерации описания объявления;
- кнопка оценки рыночной цены;
- передача в AI контекста текущего объявления: `itemId`, категория, название, цена, описание и параметры;
- обработка ошибок AI и отображение ответа в интерфейсе.

На backend для этого добавлены endpoints:
- `POST /ai/generate-description`
- `POST /ai/estimate-price`

### 5. Черновик формы
- черновик редактирования сохраняется в `localStorage`;
- ключ хранится в формате `ad-edit-draft:{id}`;
- если пользователь не печатает `1` секунду, форма автоматически сохраняется;
- при повторном открытии объявления черновик восстанавливается;
- после успешного сохранения черновик очищается.

## Структура запуска

Проект состоит из двух частей:
- frontend: текущая папка `avito`
- backend: соседняя папка `../avito-back/server`

`docker-compose.yml` лежит в корне фронтенда и поднимает сразу оба сервиса:
- frontend: `http://localhost:5173`
- backend: `http://localhost:8080`

## Локальный запуск без Docker

### 1. Backend

Перейти в папку бэка:

```powershell
cd ..\avito-back\server
```

Установить зависимости:

```powershell
npm install
```

Запустить backend:

```powershell
npm start
```

Backend будет доступен на `http://localhost:8080`.

### 2. Frontend

Вернуться в папку фронтенда:

```powershell
cd ..\avito
```

Установить зависимости:

```powershell
npm install
```

Запустить frontend в dev-режиме:

```powershell
npm run dev
```

Frontend будет доступен на `http://localhost:5173`.

### Production-like запуск фронтенда

Собрать проект:

```powershell
npm run build
```

Локально проверить собранную версию:

```powershell
npm run preview
```

Примечание: у фронтенда в `package.json` нет скрипта `npm start`, поэтому для локального запуска используются `npm run dev` и `npm run preview`.

## Запуск через Docker Compose

Из корня фронтенд-проекта:

```powershell
docker compose up --build
```



Остановка:

```powershell
docker compose down
```

После запуска:
- frontend: `http://localhost:5173`
- backend: `http://localhost:8080`

## Переменные окружения backend

Backend ожидает файл `.env` в папке `..\avito-back\server`.

Пример:

```env
PORT=8080
HOST=0.0.0.0

AI_API_BASE_URL=https://api.openai.com/v1/
AI_API_KEY=your_api_key
AI_MODEL=gpt-4.1-mini
AI_GENERATE_DESCRIPTION_PATH=/chat/completions
AI_ESTIMATE_PRICE_PATH=/chat/completions
```

Если AI-переменные не заданы, основные страницы приложения продолжают работать, но AI-функции генерации описания и оценки цены будут возвращать ошибку конфигурации.

## API, используемое фронтендом

### Объявления
- `GET /items`
- `GET /items/:id`
- `PUT /items/:id`

### AI
- `POST /ai/generate-description`
- `POST /ai/estimate-price`

## Что важно учитывать

- AI-логика вынесена на backend, чтобы не хранить API-ключ на клиенте.
- Сортировка по цене поддерживается и в UI, и в backend API.
- Черновик хранится отдельно для каждого объявления.
